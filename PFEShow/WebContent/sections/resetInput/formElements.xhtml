Click on the "Submit" button triggers a validation. The validation state and value will be saved. Simple try to skip input or input any characters with length less than 5 and click on "Submit".

Now if you would like to update server-side input values backed by managed bean, the validation state will not be reseted. This behavior can be replicated by button / select menu without attached pe:resetInput. You will see red borders around input fields and last input values passed validation. Attached pe:resetInput allows to reset the saved state of input fields completely. ${showcase:getFileContent('/sections/resetInput/example-formElements.xhtml')} ${showcase:getFileContent('/org/primefaces/extensions/showcase/controller/resetInput/ResetInputController.java')}
